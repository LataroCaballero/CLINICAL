In `backend/src/modules/presupuestos/presupuestos.service.ts`, the `marcarEnviado()` method transitions a presupuesto to ENVIADO but never sets `tokenAceptacion`. This means presupuestos marked as sent manually have no public URL.

Fix: add `import * as crypto from 'crypto';` at the top of the file (after existing imports).

In `marcarEnviado()`, change the presupuesto update data block from:
```ts
data: {
  estado: EstadoPresupuesto.ENVIADO,
  fechaEnviado: new Date(),
},
```

To:
```ts
data: {
  estado: EstadoPresupuesto.ENVIADO,
  fechaEnviado: new Date(),
  tokenAceptacion: presupuesto.tokenAceptacion ?? crypto.randomUUID(),
},
```

This requires fetching `tokenAceptacion` in the initial `findUnique` select. Change:
```ts
select: { id: true, pacienteId: true, estado: true },
```
To:
```ts
select: { id: true, pacienteId: true, estado: true, tokenAceptacion: true },
```

The `presupuesto` variable already holds `tokenAceptacion` after this change, so `presupuesto.tokenAceptacion ?? crypto.randomUUID()` will reuse an existing token if already set (idempotent), or generate a fresh UUID if null.

No other methods need changes — `enviarPresupuestoPorEmail()` in `presupuesto-email.service.ts` already generates and persists its own token correctly.
