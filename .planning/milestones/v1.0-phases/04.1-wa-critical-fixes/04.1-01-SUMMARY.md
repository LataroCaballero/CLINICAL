## Decisions

- `getUnreadCounts` returns `{}` for SECRETARIA (no DB lookup) — 0 badge count is correct UX, avoids expensive query for a role with no profesionalId
- `getTemplates` throws `BadRequestException` (not `NotFoundException`) for SECRETARIA — clear developer-facing error, SECRETARIA must use patient-scoped template listing
- `NEXT_PUBLIC_API_URL` removed from sendPresupuesto fallback chain — it is a frontend-only env var, undefined in Node.js backend runtime; `BACKEND_URL ?? 'http://localhost:3001'` with Logger.warn is correct
- `pacienteEmail` prop kept on Props type in SendWAMessageModal (backward compat) but not rendered
