Crear `backend/src/modules/reportes/services/crm-dashboard.service.ts` con `@Injectable()` e inyección de `PrismaService`. Implementar estos 5 métodos:

**`calcularRango(periodo: string): { inicio: Date; fin: Date }`** (método privado):
- `'semana'`: lunes de la semana actual (getDay=0→domingo, corregir con +1) hasta el domingo. Calcular: `dayOfWeek = now.getDay(); diff = now.getDate() - dayOfWeek + (dayOfWeek === 0 ? -6 : 1); inicio = new Date(now); inicio.setDate(diff); inicio.setHours(0,0,0,0); fin = new Date(inicio); fin.setDate(inicio.getDate() + 6); fin.setHours(23,59,59,999)`
- `'trimestre'`: primer día del trimestre actual hasta el último
- `'mes'` (default): primer y último día del mes actual

**`getFunnelSnapshot(profesionalId: string)`**:
- `ETAPAS_FUNNEL = [NUEVO_LEAD, TURNO_AGENDADO, CONSULTADO, PRESUPUESTO_ENVIADO, CONFIRMADO]`
- Hacer `prisma.paciente.groupBy({ by: ['etapaCRM'], where: { profesionalId }, _count: { id: true } })`
- Separar: los de `PERDIDO` van a `perdidos.total`, los del funnel van al array `etapas`
- Calcular `tasasPaso`: para cada par (i, i+1) → `etapas[i].count === 0 ? null : Math.round((etapas[i+1].count / etapas[i].count) * 100)`
- Calcular `motivosPerdida`: `prisma.paciente.groupBy({ by: ['motivoPerdida'], where: { profesionalId, etapaCRM: 'PERDIDO', motivoPerdida: { not: null } }, _count: { id: true } })`
- Retornar: `{ etapas: [{etapa, count}], tasasPaso: [{de, a, tasa}], perdidos: { total, porMotivo: [{motivo, count}] } }`

**`getKpis(profesionalId: string, periodo: string)`**:
- Calcular rango con `calcularRango(periodo)`
- `Promise.all` con 3 queries:
  1. `prisma.paciente.count({ where: { profesionalId, createdAt: { gte: inicio, lte: fin } } })` → `nuevos`
  2. `prisma.paciente.count({ where: { profesionalId, etapaCRM: 'CONFIRMADO', updatedAt: { gte: inicio, lte: fin } } })` → `confirmados`
  3. `prisma.paciente.count({ where: { profesionalId, etapaCRM: { not: 'PERDIDO' } } })` → `totalActivos`
- `tasaConversion = totalActivos > 0 ? Math.round((confirmados / totalActivos) * 1000) / 10 : 0`
- Retornar: `{ periodo: { inicio, fin }, nuevos, confirmados, totalActivos, tasaConversion }`

**`getMotivosPerdida(profesionalId: string, periodo: string)`**:
- Calcular rango con `calcularRango(periodo)`
- `prisma.paciente.groupBy({ by: ['motivoPerdida'], where: { profesionalId, etapaCRM: 'PERDIDO', updatedAt: { gte: inicio, lte: fin } }, _count: { id: true }, orderBy: { _count: { id: 'desc' } } })`
- `total = grupos.reduce((acc, g) => acc + g._count.id, 0)`
- Retornar: `{ total, motivos: grupos.map(g => ({ motivo: g.motivoPerdida ?? 'SIN_MOTIVO', count: g._count.id, porcentaje: total > 0 ? Math.round((g._count.id / total) * 100) : 0 })) }`

**`getPipelineIncome(profesionalId: string, periodo: string)`**:
- Calcular rango con `calcularRango(periodo)`
- `prisma.presupuesto.findMany({ where: { profesionalId, estado: 'ENVIADO', fechaEnviado: { gte: inicio, lte: fin }, paciente: { temperatura: 'CALIENTE' } }, select: { total: true } })`
- `total = presupuestos.reduce((acc, p) => acc + Number(p.total), 0)`
- Retornar: `{ total, count: presupuestos.length }`

**`getCoordinatorPerformance(profesionalId: string, periodo: string)`**:
- Calcular rango con `calcularRango(periodo)`
- Obtener todos los logs del período con su autor y datos de avance de etapa:
  ```typescript
  const logs = await prisma.contactoLog.findMany({
    where: { profesionalId, fecha: { gte: inicio, lte: fin } },
    select: {
      pacienteId: true,
      etapaCRMPost: true,
      registradoPorId: true,
      registradoPor: { select: { nombre: true } },
    },
  });
  ```
- Agrupar por `registradoPorId` (null → key `'sin-asignar'`, nombre `'Sin asignar'`):
  ```typescript
  const byCoord = new Map<string, { nombre: string; interacciones: number; pacientes: Set<string>; avances: number }>();
  for (const log of logs) {
    const key = log.registradoPorId ?? 'sin-asignar';
    const nombre = log.registradoPor?.nombre ?? 'Sin asignar';
    if (!byCoord.has(key)) byCoord.set(key, { nombre, interacciones: 0, pacientes: new Set(), avances: 0 });
    const coord = byCoord.get(key)!;
    coord.interacciones++;
    coord.pacientes.add(log.pacienteId);
    if (log.etapaCRMPost != null) coord.avances++;
  }
  ```
- Calcular `% Conversión personal = avances / pacientesContactados * 100` (null si pacientesContactados === 0):
  ```typescript
  const coordinadores = Array.from(byCoord.entries()).map(([, c]) => ({
    nombre: c.nombre,
    interacciones: c.interacciones,
    pacientesContactados: c.pacientes.size,
    porcentajeConversion: c.pacientes.size === 0 ? null : Math.round((c.avances / c.pacientes.size) * 100),
  }));
  ```
- Retornar: `{ periodo: { inicio, fin }, coordinadores }`
- Nota: `etapaCRMPost != null` indica que ese log registró un avance de etapa (la secretaria actualizó la etapa del paciente en esa interacción). Si la clínica no registra avances, `porcentajeConversion` será 0 para todos, lo cual es correcto y no es un bug.
