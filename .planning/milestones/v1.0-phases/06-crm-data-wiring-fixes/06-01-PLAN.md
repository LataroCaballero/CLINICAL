Hay un único lugar donde se llama a `createContacto()` en el controller y un único lugar donde se escribe el `contactoLog.create()` en el service. Ambos se modifican quirúrgicamente.

**1. pacientes.controller.ts — handler POST /:id/contactos**

Localizar el handler que llama a `this.pacientesService.createContacto(pacienteId, profesionalId, dto)`. Inmediatamente antes de esa llamada, extraer el userId del request:

```typescript
const registradoPorId = req.user?.userId as string | undefined;
return this.pacientesService.createContacto(pacienteId, profesionalId, dto, registradoPorId);
```

`req.user` es el payload inyectado por `JwtStrategy.validate()` que ya incluye `userId: usuario.id`. El handler ya recibe `@Req() req` o usa `@GetUser()` — verificar cuál usa el handler y extraer de allí. Si usa el decorador `@GetUser()` custom, inspeccionar su definición para saber qué expone. El patrón confirmado en otros controllers del repo es `req.user.userId`.

NOTA: El controller ya tiene lógica multi-path para `profesionalId` (PROFESIONAL vs SECRETARIA via fallback DB lookup — implementado en Phase 2.1). NO modificar esa lógica. Solo agregar la extracción de `userId` como variable separada y pasarla como 4to argumento.

**2. pacientes.service.ts — método createContacto()**

Agregar `registradoPorId?: string` como 4to parámetro del método (después de `dto: CreateContactoDto`). Dentro del bloque `$transaction`, en el objeto `data` del `tx.contactoLog.create()`, agregar:

```typescript
registradoPorId: registradoPorId ?? null,
```

La firma resultante:
```typescript
async createContacto(
  pacienteId: string,
  profesionalId: string,
  dto: CreateContactoDto,
  registradoPorId?: string,
) {
```

El campo `registradoPorId` ya existe en el schema Prisma y en la migración aplicada `20260302000000_contactolog_registrado_por`. No se requiere ninguna migración adicional.

Anti-patrón a evitar: NO pasar `req` al service. NO poner `registradoPorId` en el DTO. El service recibe solo valores primitivos, el controller extrae del request.
