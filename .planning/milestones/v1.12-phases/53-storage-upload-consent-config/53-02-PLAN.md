<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated staff → POST consent PDF | Multipart upload of arbitrary bytes claiming to be a PDF |
| request body/query → profesionalId scope | Tenant scope must come from JWT/getProfesionalId, never the body |
| uploaded buffer → disk | Untrusted content written to the uploads tree |
| request body → indicacionesUrl | User-supplied URL stored and later surfaced to patients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-06 | Tampering | consent upload content type | mitigate | Magic-byte check (`%PDF-`) on the buffer, not client mimetype; non-PDF → 400 before save, so nothing persists (D-14, INFRA-03) |
| T-53-07 | Denial of Service | consent upload size | mitigate | FileInterceptor limits.fileSize 10MB (D-11) rejects oversized uploads; global throttler from 53-01 still applies |
| T-53-08 | Elevation of Privilege | cross-tenant zona access | mitigate | getProfesionalId resolves scope from JWT; service ownership guard (zona.profesionalId !== profesionalId → NotFoundException); profesionalId never from body |
| T-53-09 | Repudiation | consent history overwrite | mitigate | Versioning: prior vigente set false, new row created; old files never deleted (D-05) — preserves legally-relevant prior consents |
| T-53-10 | Information Disclosure | stored filename leakage | mitigate | UUID filenames (D-13); client name kept only as nombreOriginal metadata, not in the served path |
| T-53-11 | Tampering | indicacionesUrl injection | mitigate | @IsUrl + @MaxLength(2048) validation; rendered as a link, not executed |
| T-53-SC | Tampering | npm installs | accept | No new package installed (multer ships with @nestjs/platform-express, throttler wired in 53-01) — no package legitimacy gate needed |
</threat_model>
