## v1 Requirements

### Embudo — estructura y clasificación

- [x] **EMBUDO-01**: La columna "Sin clasificar" aparece al final del embudo (no al inicio) y contiene solo leads sin etapa CRM real (no pacientes operados) — Phase 58 ✓
- [x] **EMBUDO-02**: El embudo muestra una nueva etapa "Cirugía Realizada" ubicada después de "Confirmado" — Phase 57 ✓
- [x] **EMBUDO-03**: Un paciente operado (fecha de cirugía pasada, o con turno de cirugía) que tiene al menos un paso pendiente aparece en "Cirugía Realizada" con indicador visual naranja — Phase 58 ✓ (browser UAT diferido)
- [x] **EMBUDO-04**: Un paciente operado con todos los pasos completos (entrada de HC + turno de cirugía con fecha + presupuesto enviado + consentimiento/indicaciones preop) se oculta del board del embudo — Phase 58 ✓ (browser UAT diferido; ver W-1: seam de consent/indicaciones)
- [x] **EMBUDO-05**: Un paciente que vuelve a solicitar un turno (u otra acción posterior) puede volver a una etapa anterior del embudo sin ser bloqueado por el guard forward-only — Phase 57 ✓

### Etiquetas de contacto en Confirmado

- [x] **CONTACTO-01**: Un paciente en "Confirmado" sin turno de cirugía muestra en la tarjeta el registro de contacto como "Espera fecha" — Phase 58 ✓ (browser UAT diferido)
- [x] **CONTACTO-02**: Un paciente en "Confirmado" con turno de cirugía muestra en la tarjeta el registro de contacto como "Cirugía programada" — Phase 58 ✓ (browser UAT diferido)

### Stepper accionable (sheet lateral CRM)

- [x] **STEPPER-01**: Cada paso del stepper que no tiene pendientes se muestra con círculo verde y sin botón de quick-action — Phase 59 ✓
- [x] **STEPPER-02**: Cada paso del stepper que tiene algo pendiente se muestra con círculo naranja — Phase 59 ✓
- [x] **STEPPER-03**: Al hacer click en un paso naranja de entrada de HC, se abre el wizard de HC para seleccionar la plantilla pertinente y cargar la entrada — Phase 59 ✓
- [x] **STEPPER-04**: Al hacer click en un paso naranja de presupuesto, se abre el modal de presupuesto prellenado con los tratamientos/cirugías cargados en la ficha del paciente — Phase 59 ✓
- [x] **STEPPER-05**: En el paso "Confirmado", si el paciente no tiene turno de cirugía, el paso se muestra pendiente ("fecha") y al hacer click se abre la agenda para programar el turno de cirugía — Phase 59 ✓
- [x] **STEPPER-06**: En el paso "Confirmado", si el paciente tiene turno de cirugía, el paso se muestra verde — Phase 59 ✓

### Estadísticas sobre registros reales

- [x] **STATS-01**: El conteo de cirugías realizadas se calcula sobre registros efectivamente realizados (Cirugia/HC), robusto a que la etapa CRM del paciente cambie después — Phase 60 ✓ (backend 8/8 + SECURED; display verificado)
- [x] **STATS-02**: El conteo de tratamientos realizados se calcula sobre registros efectivamente realizados (HC de tratamiento), robusto a que la etapa CRM del paciente cambie después — Phase 60 ✓ (backend 8/8 + SECURED; display verificado)
