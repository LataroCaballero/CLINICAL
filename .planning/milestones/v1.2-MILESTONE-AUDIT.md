### Known Gap (Tech Debt)

**CAE-03 BUG-1 — `afipError` never persisted for UnrecoverableError** (business rejection path)

`CaeEmissionProcessor.onFailed` (processor.ts ~line 65):
```
if (job.attemptsMade >= maxAttempts) {
  await prisma.factura.update({ data: { afipError: job.failedReason } });
  await caeaService.asignarCaeaFallback(...)
}
```
For `UnrecoverableError` (AFIP error 10242, resultado=R): BullMQ moves the job to failed immediately. `onFailed` fires with `job.attemptsMade = 1`. Since `1 < maxAttempts (5)`, the guard is false — `afipError` is never written. The Facturador sees the invoice silently stuck in `EMISION_PENDIENTE`.

**Fix:** Persist `afipError` unconditionally in `onFailed`, regardless of `attemptsMade`:
```ts
await prisma.factura.update({ data: { afipError: job.failedReason } });
if (job.attemptsMade >= maxAttempts) {
  await caeaService.asignarCaeaFallback(...)
}
```

---

**CAE-03 BUG-2 — Modal condition wrong after CAEA fallback** (transient exhaustion path)

`FacturaDetailModal.tsx` ~line 270:
```tsx
{factura?.afipError && factura.estado === EstadoFactura.EMISION_PENDIENTE && (
  <ErrorPanel ... />
)}
```
After 5 transient retries: `onFailed` fires, `afipError` IS written, then `asignarCaeaFallback` sets `estado = CAEA_PENDIENTE_INFORMAR`. The condition `estado === EMISION_PENDIENTE` is now false → error panel hidden.

**Fix:** Expand modal condition to also show panel when `estado === CAEA_PENDIENTE_INFORMAR`:
```tsx
{factura?.afipError && (
  factura.estado === EstadoFactura.EMISION_PENDIENTE ||
  factura.estado === EstadoFactura.CAEA_PENDIENTE_INFORMAR
) && (
  <ErrorPanel ... />
)}
```
