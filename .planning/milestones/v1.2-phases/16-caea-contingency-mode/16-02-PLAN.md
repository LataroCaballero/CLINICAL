    Constructor: PrismaService, CaeaService, ConfigService, Logger

    @Cron('0 6 12,27 * *')
    async prefetchAllTenants(): Promise<void>
    - Load all ConfiguracionAFIP: prisma.configuracionAFIP.findMany({})
    - For each: compute { periodo, orden } = calcularProximoPeriodoYOrden(new Date())
    - Call caeaService.solicitarYPersistir(cfg.profesionalId, periodo, orden) inside try/catch
    - On error: logger.error('CAEA pre-fetch failed for profesionalId...') and continue (never throw)
    - After pre-fetch loop: call checkDeadlines() (same cron run)

    @Cron('0 7 * * *') — separate daily check for deadline alerts (or called from prefetchAllTenants)
    Implementation note: Call checkDeadlines() from within prefetchAllTenants() directly (no separate cron needed — Research CAEA-04 pattern says "same bimensual cron").

    async checkDeadlines(): Promise<void>
    - Load all CaeaVigente with profesional.usuario include: prisma.caeaVigente.findMany({ include: { profesional: { include: { usuario: true, configClinica: true } } } })
    - For each caeaVigente:
      - Parse fchTopeInf: year=parseInt(fch.slice(0,4)), month=parseInt(fch.slice(4,6))-1, day=parseInt(fch.slice(6,8)); deadlineDate = new Date(year, month, day)
      - daysUntilDeadline = Math.ceil((deadlineDate.getTime() - Date.now()) / 86400000)
      - if daysUntilDeadline > 2: skip
      - pendingCount = await prisma.factura.count({ where: { profesionalId: caeaVigente.profesionalId, estado: 'CAEA_PENDIENTE_INFORMAR' } })
      - if pendingCount === 0: skip
      - sendDeadlineAlert(caeaVigente, pendingCount, daysUntilDeadline)

    private async sendDeadlineAlert(caeaVigente, pendingCount, daysLeft): Promise<void>
    - Follow exact nodemailer pattern from CertExpiryScheduler (SMTP config from ConfigService)
    - Subject: `URGENTE: CAEA vence en ${daysLeft} días — ${pendingCount} facturas sin informar`
    - Body: HTML with CUIT, fchTopeInf, pendingCount, link to admin panel

    Modified onFailed:
    ```typescript
    @OnWorkerEvent('failed')
    async onFailed(job: Job<CaeJobData>): Promise<void> {
      this.logger.error(`CAE job ${job.id} failed: ${job.failedReason}`);
      const maxAttempts = job.opts.attempts ?? 3;
      if (job.attemptsMade >= maxAttempts) {
        this.logger.warn(`Max retries reached for facturaId ${job.data.facturaId} — attempting CAEA fallback`);
        await this.caeaService.asignarCaeaFallback(job.data.facturaId, job.data.profesionalId);
      }
    }
    ```

    Note: onFailed becomes async. CaeaService.asignarCaeaFallback handles its own null guard, so no try/catch needed here — any unexpected error from the fallback should be visible in logs.
