# Milestone v1.3: Historial de Consultas

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 20–21
**Total Plans:** 4

## Overview

Expansión del widget "Turnos del día" para que el profesional pueda navegar a cualquier día, ver la agenda completa con métricas, y agregar entradas de HC retroactivas a turnos finalizados usando el mismo formato que LiveTurno.

## Phases

### Phase 20: Backend Data Fixes

**Goal:** Los endpoints de turnos exponen todos los campos que el frontend necesita, y el backend acepta entradas HC con fecha histórica
**Depends on:** Nothing (first phase of v1.3)
**Plans:** 1 plan

Plans:
- [x] 20-01: Fixes quirúrgicos en turnos.service.ts y soporte fecha retroactiva en HC

**Details:**
- `obtenerAgendaDiaria` selects expanded to include `diagnostico` and `tratamiento` from patient
- `obtenerProximosTurnos` selects expanded to include `esCirugia` and `entradaHCId` at turno level and `esCirugia` in `tipoTurno`
- `CreateEntradaDto` gained optional `fecha` field with `@IsOptional` + `@IsDateString`; service validates and rejects future dates with 400; persists retroactive date via conditional spread

### Phase 21: Agenda Widget + Modal HC

**Goal:** El profesional puede navegar día a día desde el dashboard, ver la agenda completa con métricas, y agregar entradas HC retroactivas a cualquier turno finalizado
**Depends on:** Phase 20
**Plans:** 3 planes

Plans:

- [x] 21-02: UpcomingAppointments agenda-first: hoy por defecto, nav día-a-día, métricas, Ver HC

**Details:**

- Plan 21-03: `fecha: yyyyMmDd(selectedDate)` wired into `createTemplateEntry.mutateAsync` DTO in `handleSelectTemplate`; `EntryReadOnly` Lock badge confirmed present for legal immutability indicator

## Milestone Summary

**Key Decisions:**
- Future date boundary uses `hoy.setHours(23, 59, 59, 999)` so today is accepted (not rejected as future)
- `fechaFinal` only spread into Prisma when provided; DB `@default(now())` handles nil case without behavior regression
- Single `selectedDate: Date` state replaces dual `dateIndex` + `pickedDate` — eliminates mode-switching complexity
- `isHoyOPasado = isToday || isPast` is the single gate for both metrics and "Ver HC" button visibility
- Retroactive fecha pattern from Phase 20 reused verbatim in Phase 21 for hc-templates endpoint

**Issues Resolved:**
- Prisma select field gaps causing missing `esCirugia`/`entradaHCId` in frontend response
- Dual-mode architecture in UpcomingAppointments causing picker/nav state mismatch
- HC entries always timestamped to "now" instead of historical turno date

**Issues Deferred:**
- Human-verify checkpoint (Task 2 of plan 21-03) — pending browser E2E test by user

**Technical Debt Incurred:**
- None net-new (existing tech debt carries forward from v1.2)
