export function useUpdateListaEspera() {

**Part A — Create CambiarFlujoModal.tsx:**

Create `frontend/src/app/dashboard/pacientes/components/CambiarFlujoModal.tsx`.

Props interface:
```typescript
interface CambiarFlujoModalProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  pacienteId: string;
  currentFlujo: "CIRUGIA" | "TRATAMIENTO" | "PENDIENTE" | null;
  onOptimisticUpdate: (flujo: "CIRUGIA" | "TRATAMIENTO" | "PENDIENTE") => void;
  onRevert: () => void;
}
```

Implementation:
1. Use shadcn `Dialog`, `DialogContent`, `DialogHeader`, `DialogTitle`, `DialogFooter` (same imports as `HCCreatorDialog.tsx`).

2. Local state: `const [selectedFlujo, setSelectedFlujo] = useState<"CIRUGIA" | "TRATAMIENTO" | "PENDIENTE">(currentFlujo ?? "PENDIENTE")`. Reset on open: `useEffect(() => { if (open) setSelectedFlujo(currentFlujo ?? "PENDIENTE"); }, [open, currentFlujo])`.

3. Call `useUpdateFlujo(pacienteId)` to get the mutation.

4. Call `useRouter()` from `next/navigation` for toast navigation.

5. Three selector cards — one for each flujo value. Use exact FlujoBadge color classes:
   - CIRUGIA: base `bg-blue-100 text-blue-700`, selected ring `ring-2 ring-blue-400`
   - TRATAMIENTO: base `bg-green-100 text-green-700`, selected ring `ring-2 ring-green-400`
   - PENDIENTE: base `bg-amber-100 text-amber-700`, selected ring `ring-2 ring-amber-400`
   
   Card labels: "Cirugía", "Tratamiento", "Pendiente" (full words, not the badge abbreviations).
   Each card is a `<button>` with `onClick={() => setSelectedFlujo(value)}`.

6. Below the cards, add a muted description text:
   `"Al confirmar, el paciente quedará en la etapa Sin Clasificar y se registrará un contacto automático."`

7. DialogFooter with two buttons:
   - "Cancelar" → `onOpenChange(false)`
   - "Confirmar" → disabled when `selectedFlujo === currentFlujo || mutation.isPending`. Shows loading state text "Guardando..." when `mutation.isPending`.

8. `handleConfirmar`:
   ```typescript
   const handleConfirmar = () => {
     if (!selectedFlujo || selectedFlujo === currentFlujo) return;
     onOptimisticUpdate(selectedFlujo);
     onOpenChange(false);
     mutation.mutate(selectedFlujo, {
       onError: () => {
         onRevert();
         toast.error("Error al actualizar el flujo. Intentá nuevamente.");
       },
       onSuccess: () => {
         const label = { CIRUGIA: "Cirugía", TRATAMIENTO: "Tratamiento", PENDIENTE: "Pendiente" }[selectedFlujo];
         toast(`Flujo actualizado a ${label}`, {
           action: {
             label: "Ver en CRM →",
             onClick: () => {
               localStorage.setItem("pacientes-vista", "embudo");
               router.refresh();
             },
           },
         });
       },
     });
   };
   ```

   IMPORTANT: `onOptimisticUpdate` and `onOpenChange(false)` fire BEFORE `mutation.mutate` so the UI updates and modal closes immediately — this is the optimistic pattern.

   IMPORTANT for toast navigation: `localStorage.setItem("pacientes-vista", "embudo")` + `router.refresh()` — the page reads localStorage on mount via `useState(() => localStorage.getItem(STORAGE_KEY))`, and `router.refresh()` triggers a server component re-render causing the page state to reinitialize from localStorage.
