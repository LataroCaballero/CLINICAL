  <action>
In `obtenerTurnosPorRango` (currently ends at line ~407 with `return this.prisma.turno.findMany(...)`), replace the direct `return` with a two-step approach:

1. Assign the Prisma result to `const turnos` instead of returning directly.
2. After the findMany, collect unique patient IDs:
   ```typescript
   const pacienteIds = [...new Set(turnos.map((t) => t.paciente.id))];
   ```
3. Run ONE batch query for HC histories (use `take: 10` on nested entradas to limit volume per patient):
   ```typescript
   const historias = await this.prisma.historiaClinica.findMany({
     where: { pacienteId: { in: pacienteIds } },
     select: {
       pacienteId: true,
       entradas: {
         orderBy: { fecha: 'desc' },
         take: 10,
         select: { contenido: true },
       },
     },
   });
   ```
4. Build lookup Map with JS filtering (do NOT use Prisma JSON `path equals` — it is not supported and throws runtime error):
   ```typescript
   const ultimoTratamientoMap = new Map<string, string | null>();
   for (const historia of historias) {
     const lastEntry = historia.entradas.find((e) => {
       const c = e.contenido as Record<string, unknown> | null;
       return Array.isArray(c?.tratamientos) && (c!.tratamientos as unknown[]).length > 0;
     });
     if (lastEntry) {
       const tratamientos = (lastEntry.contenido as { tratamientos: Array<{ nombre: string }> }).tratamientos;
       ultimoTratamientoMap.set(historia.pacienteId, tratamientos.map((t) => t.nombre).join(', '));
     } else {
       ultimoTratamientoMap.set(historia.pacienteId, null);
     }
   }
   ```
5. Return mapped result:
   ```typescript
   return turnos.map((t) => ({
     ...t,
     ultimoTratamiento: ultimoTratamientoMap.get(t.paciente.id) ?? null,
   }));
   ```

Edge cases:
- Empty `pacienteIds` (no turnos in range): skip the historias query entirely with an early return of `[]`.
- Patient with no HistoriaClinica record at all: not in `historias` array, `Map.get` returns `undefined`, `?? null` handles it.
- `contenido` is null (older entries): guarded by `c?.tratamientos` optional chain.
