<objective>
Implementar el guard forward-only en todos los puntos de auto-transición CRM del sistema para que las decisiones manuales del profesional no sean revertidas por eventos automáticos.

Purpose: Si un profesional mueve manualmente a un paciente a CONFIRMADO y luego el paciente hace click en un link viejo de presupuesto, su etapa no debe retroceder. El guard garantiza que las auto-transiciones del sistema solo avanzan, nunca retroceden.

Output: Constante `ETAPA_ORDEN` y función `isAutoTransitionBlocked` definidas en `presupuestos.service.ts` y `turnos.service.ts` (como module-level helpers), aplicadas en 5 call sites de auto-transición.
</objective>

```typescript
// Constante de orden CRM — PERDIDO excluido intencionalmente (manejo especial)
const ETAPA_ORDEN: Record<string, number> = {
  SIN_CLASIFICAR: 0,
  NUEVO_LEAD: 1,
  TURNO_AGENDADO: 2,
  CONSULTADO: 3,
  PRESUPUESTO_ENVIADO: 4,
  CONFIRMADO: 5,
  PROCEDIMIENTO_REALIZADO: 6,
};

function etapaOrden(e: EtapaCRM | null | undefined): number {
  return ETAPA_ORDEN[e ?? 'SIN_CLASIFICAR'] ?? 0;
}

// Returns true when the auto-transition should be SKIPPED
function isAutoTransitionBlocked(
  actual: EtapaCRM | null | undefined,
  destino: EtapaCRM,
): boolean {
  return etapaOrden(actual) >= etapaOrden(destino);
}
```
