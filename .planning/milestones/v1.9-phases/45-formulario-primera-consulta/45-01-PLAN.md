Nueva forma de selección agrupada por zona (DTO y contenido JSONB):
```typescript
// Un item de zona seleccionada
interface ZonaSeleccionDto {
  zonaId: string;        // ZonaHC.id del catálogo (referencia; el snapshot son los nombres)
  zona: string;          // nombre de la zona, ej. "Abdomen"
  diagnosticos: string[]; // nombres de diagnósticos seleccionados (snapshot)
  otroTexto?: string;    // texto libre cuando la zona es "Otros" (paridad con comportamiento actual)
  tratamientos: { nombre: string; tratamientoId?: string; precio: number }[];
}
```

Contenido JSONB resultante para entradas nuevas de primera_vez:
```json
{
  "tipo": "primera_vez",
  "zonas": [ /* ZonaSeleccionDto[] */ ],
  "comentario": "",
  "presupuestoId": null,
  "presupuestoTotal": 0
}
```

Forma legacy (entradas históricas y fallback — NO se migra, ver REQUIREMENTS Out of Scope):
```json
{
  "tipo": "primera_vez",
  "diagnostico": { "zonas": [], "subzonas": [], "otroTexto": "" },
  "tratamientos": [ { "nombre": "", "tratamientoId": "", "precio": 0 } ],
  "comentario": "", "presupuestoId": null, "presupuestoTotal": 0
}
```

Estado actual del service (historia-clinica.service.ts líneas 83-120): construye `contenido` inline para primera_vez desde `dto.diagnostico`/`dto.tratamientos`, y deriva `diagnosticoStr`/`tratamientoStr` para actualizar el perfil del paciente (líneas 238-245 hacen el update con esos strings).
