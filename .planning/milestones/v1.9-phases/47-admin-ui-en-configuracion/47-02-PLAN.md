<task type="auto">
  <name>Task 1: Hook useCatalogoHCMutations (renombrar + eliminar)</name>
  <files>frontend/src/hooks/useCatalogoHCMutations.ts</files>
  <action>
    Crear el hook de mutaciones siguiendo el patrón EXACTO de useTratamientosProfesional.ts.
    Importar useMutation, useQueryClient de '@tanstack/react-query', api de '@/lib/api', y
    CATALOGO_HC_QUERY_KEY de '@/hooks/useCatalogoHC'.

    Exportar 6 hooks, cada uno recibe `profesionalId?: string`:
    - useRenombrarZona(profesionalId?)        → api.patch(`/catalogo-hc/zonas/${id}`, { nombre })
    - useEliminarZona(profesionalId?)          → api.delete(`/catalogo-hc/zonas/${id}`)
    - useRenombrarDiagnostico(profesionalId?)  → api.patch(`/catalogo-hc/diagnosticos/${id}`, { nombre })
    - useEliminarDiagnostico(profesionalId?)   → api.delete(`/catalogo-hc/diagnosticos/${id}`)
    - useRenombrarTratamiento(profesionalId?)  → api.patch(`/catalogo-hc/tratamientos/${id}`, { nombre })
    - useEliminarTratamiento(profesionalId?)   → api.delete(`/catalogo-hc/tratamientos/${id}`)

    Las mutaciones de renombrar reciben `{ id: string; nombre: string }`; las de eliminar reciben `id: string`.
    Todas pasan `{ params: profesionalId ? { profesionalId } : {} }` a axios.
    Todas en onSuccess: `queryClient.invalidateQueries({ queryKey: [CATALOGO_HC_QUERY_KEY] })`
    (invalida por prefijo todas las variantes cacheadas — patrón establecido en Phase 46).
  </action>
  <verify>
    <automated>cd frontend && npx tsc --noEmit 2>&1 | grep -i "useCatalogoHCMutations" || echo "OK: hook compila"</automated>
  </verify>
