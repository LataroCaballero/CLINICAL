**whatsapp-webhook.controller.ts** — NEW file, no @Auth decorator (public endpoints):
```typescript
import { Controller, Get, Post, Query, Body, Res, HttpCode, Logger } from '@nestjs/common';
import { InjectQueue } from '@nestjs/bullmq';
import { Queue } from 'bullmq';
import { Response } from 'express';
import { WHATSAPP_QUEUE } from './processors/whatsapp-message.processor';
import { PrismaService } from '../../prisma/prisma.service';
import { EtapaCRM, DireccionMensajeWA, TipoMensajeWA, TipoContacto } from '@prisma/client';

@Controller()
export class WhatsappWebhookController {
  private readonly logger = new Logger(WhatsappWebhookController.name);

  constructor(
    @InjectQueue(WHATSAPP_QUEUE) private readonly whatsappQueue: Queue,
    private readonly prisma: PrismaService,
  ) {}

  @Get('webhook/whatsapp')
  verifyWebhook(@Query() query: any, @Res() res: Response) {
    const mode = query['hub.mode'];
    const token = query['hub.verify_token'];
    const challenge = query['hub.challenge'];
    if (mode === 'subscribe' && token === process.env.WA_WEBHOOK_VERIFY_TOKEN) {
      this.logger.log('WhatsApp webhook verified');
      return res.status(200).send(challenge);
    }
    this.logger.warn('WhatsApp webhook verification failed');
    return res.status(403).send('Forbidden');
  }

  @Post('webhook/whatsapp')
  @HttpCode(200) // ALWAYS return 200 first — Meta requires this
  async handleWebhook(@Body() body: any): Promise<string> {
    // Enqueue async — never block here
    await this.whatsappQueue.add('process-webhook', body, {
      attempts: 3,
      backoff: { type: 'exponential', delay: 1000 },
    });
    return 'OK';
  }
}
```

**processors/whatsapp-message.processor.ts** — add `process-webhook` case and implement `send-whatsapp-message`. Import PrismaService (inject it — add to constructor). The processor already extends WorkerHost:

For `send-whatsapp-message` case:
- job.data = {mensajeId: string} (the DB record ID was created before enqueueing)
- Fetch MensajeWhatsApp by id. If estado is not PENDIENTE, skip (already processed).
- Fetch ConfiguracionWABA for profesionalId. Decrypt accessToken.
- Fetch paciente.telefono. Normalize phone.
- POST to Meta API (for template: use contenido as templateName with languageCode 'es')
- On success: prisma.mensajeWhatsApp.update({estado: ENVIADO, waMessageId: res.data.messages[0].id, sentAt: new Date()})
- On error: prisma.mensajeWhatsApp.update({estado: FALLIDO, errorMsg: e.message})

For `process-webhook` case:
- Parse Meta webhook body: entry[0].changes[0].value
- If value.statuses exists: for each status in statuses[], upsert MensajeWhatsApp by waMessageId:
  - 'sent' → ENVIADO, sentAt=new Date()
  - 'delivered' → ENTREGADO, deliveredAt=new Date()
  - 'read' → LEIDO, readAt=new Date()
  - 'failed' → FALLIDO, errorMsg=status.errors?.[0]?.message
  - Use prisma.mensajeWhatsApp.updateMany({where:{waMessageId:status.id}, data:{estado}}) — dedup key
- If value.messages exists: for each msg in messages[] (inbound from patient):
  - Find paciente where telefono stripped matches msg.from (use prisma.paciente.findFirst({where:{telefono:{contains:msg.from.slice(-8)}}}))
  - If paciente found:
    1. Create MensajeWhatsApp: {pacienteId, waMessageId: msg.id, tipo: CUSTOM, contenido: msg.text?.body ?? '[media]', estado: ENTREGADO, direccion: INBOUND, profesionalId: paciente.profesionalId}
    2. Create ContactoLog: {pacienteId, profesionalId: paciente.profesionalId, tipo: MENSAJE, nota: `WhatsApp inbound: ${msg.text?.body?.slice(0,100) ?? '[media]'}`}
    3. Auto-heat temperatura: prisma.paciente.update({where:{id:paciente.id}, data:{temperatura: 'CALIENTE'}}) — CRM-05
    4. Update paciente.hasUnreadWA = true (skip if field doesn't exist — check schema first)
