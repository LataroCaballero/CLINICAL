  <how-to-verify>
**Setup (if not already running):**
- Backend: `cd backend && npm run start:dev`
- Frontend: `cd frontend && npm run dev` → http://localhost:3000

**Test 1 — QR-01: PDF with QR (requires an EMITIDA invoice with cae set)**
1. Navigate to Finanzas → Comprobantes tab
2. If no EMITIDA invoice with cae exists: create one via stub mode (POST /finanzas/facturas/:id/emitir) or use the dev DB
3. Click the Download button (cloud/arrow icon) on an EMITIDA invoice row
4. Open the downloaded PDF
5. Confirm: PDF opens, contains a QR code image in the footer/bottom section
6. Scan the QR with a phone or use an online QR decoder
7. Confirm: QR decodes to URL starting with https://www.afip.gob.ar/fe/qr/?p=
8. Paste the URL in browser — confirm AFIP page loads (may show validation info)

**Test 2 — QR-02: CAE display in FacturaDetailModal**
1. Click on any EMITIDA invoice row in the table
2. Confirm: Detail modal opens
3. Confirm: CAE number is visible (14-digit string in monospace font)
4. Confirm: CAE vencimiento date is visible (formatted DD/MM/YYYY)
5. Confirm: QR image is rendered (small square barcode image visible — NOT a broken img tag)
6. Confirm: "Escanear para verificar en AFIP" label is present below QR image

**Test 3 — QR-02: Estados en tabla**
1. Confirm: EMISION_PENDIENTE shows "Emitiendo..." badge in amber if such invoice exists
2. If no EMISION_PENDIENTE invoice exists — verify by checking ESTADO_BADGE handles it (no runtime error in browser console)

**Test 4 — QR-03: USD invoice tipoCambio**
1. In DB (or via API), identify or create a USD invoice (moneda='USD')
2. Click on it to open FacturaDetailModal
3. Confirm: Warning message about USD and cotización BNA is visible
4. Confirm: Link to https://www.bna.com.ar/Home/Cotizaciones is present and opens in new tab
5. Confirm: Input field labeled "Cotización BNA" is present
6. Enter a value (e.g. 950.50) and click Save
7. Confirm: toast success appears, close and reopen modal — confirm tipoCambio shows 950.50

**Test 5 — Non-emitted invoice Download button**
1. Click Download button on an invoice that has cae=null (EMITIDA without AFIP, or EMISION_PENDIENTE)
2. Confirm: Toast notification appears saying to emit via AFIP first (not a broken network request)
