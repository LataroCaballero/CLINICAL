Este comando:
1. Lee la nueva migración en `migrations/20260416000000_flujo_paciente/migration.sql`
2. La aplica a la BD
3. Regenera automáticamente `@prisma/client` con el nuevo enum `FlujoPaciente` y los nuevos campos

- Error `type "FlujoPaciente" already exists` → el tipo ya fue creado en una migración anterior (poco probable); revisar con `psql -c "\dT FlujoPaciente"`

- Error de unique constraint → el INSERT intentó crear 'Control' duplicado; el migration.sql debe tener UPDATE para 'Control' y solo INSERT 4 nuevos tipos
