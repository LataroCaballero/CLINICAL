<objective>
Agregar endpoints de renombrado y eliminación (soft-delete) al módulo catalogo-hc para zonas, diagnósticos y tratamientos del catálogo HC del profesional. Esto habilita que el Admin UI (plan 47-02) modifique el catálogo sin tocar código.

Purpose: Cumplir ADM-02 (renombrar) y ADM-03 (eliminar) a nivel backend. El soft-delete (`activo=false`) es el mecanismo correcto porque las HistoriaClinicaEntrada guardan el TEXTO del catálogo en JSONB `contenido` (sin FK), por lo que desactivar un ítem lo saca del formulario Primera Consulta (que ya filtra `activo: true`) sin alterar ninguna HC histórica.

Output: DTO de renombrado, 6 métodos de servicio (3 renombrar + 3 eliminar), y endpoints PATCH/DELETE en el controller, con tests unitarios.
</objective>

Schema Prisma (backend/src/prisma/schema.prisma) — los 3 modelos del catálogo HC.
Todos tienen: `activo Boolean @default(true)` y `esSistema Boolean @default(false)`.

```prisma
model ZonaHC {
  id String @id; nombre String; orden Int; activo Boolean @default(true)
  esSistema Boolean @default(false); profesionalId String
  diagnosticos DiagnosticoHC[]; tratamientos TratamientoHC[]
  @@unique([nombre, profesionalId])
}
model DiagnosticoHC {
  id String @id; nombre String; orden Int @default(0); activo Boolean @default(true)
  esSistema Boolean @default(false); zonaId String; profesionalId String
  @@unique([nombre, zonaId])  // zona ZonaHC onDelete: Cascade
}
model TratamientoHC {
  id String @id; nombre String; orden Int @default(0); activo Boolean @default(true)
  esSistema Boolean @default(false); zonaId String; profesionalId String; tratamientoId String?
  @@unique([nombre, zonaId])  // zona ZonaHC onDelete: Cascade
}
```

CRÍTICO — HistoriaClinicaEntrada NO tiene FK a estos modelos. Guarda el texto del
diagnóstico/tratamiento en su JSONB `contenido`. Por eso el borrado es soft-delete
(`activo=false`), NUNCA `prisma.*.delete()`. Las HC históricas no se tocan jamás.

CRÍTICO — `esSistema=true` marca los ítems "Otros" (la zona "Otros" y el diagnóstico/
tratamiento "Otros" de cada zona). Estos NO se pueden renombrar ni eliminar.

Patrón getProfesionalId — YA existe en catalogo-hc.controller.ts (copiado de
tratamientos.controller.ts). Resuelve profesionalId desde el JWT (PROFESIONAL) o desde
query param ?profesionalId= (SECRETARIA/ADMIN). Reutilizar ese método tal cual.
