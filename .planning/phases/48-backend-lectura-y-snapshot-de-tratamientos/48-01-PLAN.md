    - Shape v1.9 agrupado: `{ zonas: [{ zona:'Abdomen', tratamientos:[{nombre:'Lipoaspiración'}] }, { zona:'Mamas', tratamientos:[{nombre:'Mastopexia'},{nombre:'Implante'}] }] }` → aplana a través de zonas y aplica resumen-con-conteo: `"Lipoaspiración +2"` (primer nombre + cantidad restante).
    - Shape legacy plano: `{ tratamientos:[{nombre:'Abdominoplastia'}] }` → `"Abdominoplastia"`.
    - Shape legacy multi: `{ tratamientos:[{nombre:'A'},{nombre:'B'}] }` → `"A +1"`.
    - tratamiento_en_consultorio con catálogo: `{ tipo:'tratamiento_en_consultorio', tratamientos:[{nombre:'Toxina'}], texto:'nota extra' }` → catálogo gana: `"Toxina"` (ignora texto).
    - tratamiento_en_consultorio sin catálogo: `{ tipo:'tratamiento_en_consultorio', tratamientos:[], texto:'Limpieza facial profunda' }` → fallback a texto.
    - Texto libre puro largo: `{ tipo:'seguimiento', texto:'<texto > ~80 chars>' }` → recortado a ~80 chars con sufijo '…'.
    - Texto libre puro corto → sin recorte.
    - Sin tratamientos ni texto / contenido null / contenido undefined → `null`.
    - Edge: zonas vacías `{ zonas:[] }` y tratamientos vacíos `{ tratamientos:[] }` → null. Tratamientos sin `nombre` (nombre vacío/undefined) → se descartan; si quedan 0 → null.
  </behavior>
  <action>
    Escribir primero los tests en historia-clinica.contenido.spec.ts (nuevo `describe('resumirTratamientosDeContenido')`) y correrlos para confirmar RED. Luego implementar la función en el helper.

    Reglas de implementación (decisiones del CONTEXT):
    1. Normalizar contenido a `Record<string, unknown> | null`; si null/no-objeto → return null.
    2. Recolectar nombres de tratamiento según prioridad de shape:
       - Si `Array.isArray(contenido.zonas) && contenido.zonas.length > 0` → `zonas.flatMap(z => z.tratamientos ?? []).map(t => t.nombre)`.
       - Else si `Array.isArray(contenido.tratamientos)` → `contenido.tratamientos.map(t => t.nombre)`.
       - Filtrar nombres falsy/whitespace (trim) en ambos casos.
    3. Si hay >=1 nombre válido → aplicar resumen-con-conteo: si 1 → ese nombre; si N>1 → `${primero} +${N-1}`.
    4. Si NO hay nombres válidos pero existe `contenido.texto` string no vacío (trim) → recortar: si length > LIMITE (constante local, sugerido 80) → `texto.slice(0, LIMITE).trimEnd() + '…'`; si no, texto tal cual.
    5. Else → null.
    Mantener la función pura (sin imports de NestJS/Prisma). Exportarla con `export function`. Tipar los items de tratamiento de forma defensiva (`{ nombre?: unknown }`) ya que el JSONB es untyped.
    NOTA: el resumen-con-conteo aplica uniformemente a zonas, legacy y catálogo de consultorio — un único colector de nombres seguido del mismo formateador evita duplicar lógica.
