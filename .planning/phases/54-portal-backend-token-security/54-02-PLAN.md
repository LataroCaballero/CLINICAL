    - Any verify while portalBloqueadoHasta > now → throws 429 immediately (before checking DNI).
    - Block-duration model (D-03): the counter resets to 0 ONLY when a prior block has expired (portalBloqueadoHasta is SET AND < now), giving a fresh 3 attempts; it does NOT reset while portalBloqueadoHasta is null (attempts 1-2).
    - Correct DNI → resets portalIntentosFallidos to 0, clears portalBloqueadoHasta, returns a signed JWT containing { sub: pacienteId, scope: 'portal-paciente' }.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL `:token` → DB lookup | Raw UUID is attacker-controllable; never stored/compared in raw form |
| patient body → prisma update | Untrusted body must not reach curated clinical fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-04 | Information Disclosure | token lookup | mitigate | Lookup by `sha256(rawToken)` against `portalToken` hash; a DB dump never yields a usable raw token; unknown token → 404 with no patient data (D-07) |
| T-54-05 | Spoofing / brute force | DNI verify | mitigate | Persistent 3-strike lock per token (D-01/02), 15-min auto-expiry + reset-on-success (D-03) → 429; counters survive restarts/multi-instance |
| T-54-06 | Elevation / mass assignment | write methods | mitigate | Prisma `data` object built only from allowed contact / `*AutoReportad*` keys; curated `alergias`/`condiciones`/`medicacion` never present (SC#4) |
| T-54-07 | Information Disclosure | no-DNI patient | mitigate | Normalized-blank stored DNI → UnauthorizedException; blank input cannot match blank stored value |
| T-54-08 | Spoofing | portal JWT | mitigate | Short-TTL JWT carries `scope: 'portal-paciente'`; raw token is never a write credential (D-05) |
| T-54-SC | Tampering | npm installs | accept | No new package installs this phase |
</threat_model>
