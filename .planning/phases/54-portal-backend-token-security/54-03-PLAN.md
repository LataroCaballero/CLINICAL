<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → :token / :token/verificar | Unauthenticated PHI surface; rate-limited, no patient data pre-verify |
| patient body → write route | Prohibited clinical/obra-social/dni fields must be stripped before reaching the service |
| portal JWT → guarded route | Only a valid portal-scoped JWT may read/write; pacienteId never from body/URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-09 | Denial of Service | public routes | mitigate | Class-level strict `@Throttle({ default: { ttl: 60000, limit: 20 } })` tier (20/min) over the global APP_GUARD throttler |
| T-54-10 | Tampering / mass assignment | write routes | mitigate | Per-route `new ValidationPipe({ whitelist: true })` (no `forbidNonWhitelisted`) silently strips `alergias`/`condiciones`/`etapaCRM`/`flujo`/`dni`/`obraSocialId` — SC#3/D-12; defense-in-depth with service-side data confinement |
| T-54-11 | Elevation | guarded routes | mitigate | `@UseGuards(PortalJwtGuard)` per route; `pacienteId` from `req.user` only (pitfall 12); pre-verify/verify stay public |
| T-54-12 | Information Disclosure | pre-verify route | mitigate | `@Get(':token')` returns 200/404 only, no patient data (D-07) |
| T-54-SC | Tampering | npm installs | accept | No new package installs this phase |
</threat_model>
