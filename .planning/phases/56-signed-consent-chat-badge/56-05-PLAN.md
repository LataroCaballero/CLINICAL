| App → disk storage | The immutable signed PDF is archived per-tenant |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-12 | Elevation/IDOR | firmarConsentimiento identity | mitigate | `pacienteId` from JWT only; `dto.zonaId` re-validated to belong to the patient's profesional before use (pitfall E) |
| T-56-13 | Spoofing/Repudiation | Forensic IP/userAgent | mitigate | Captured server-side from request headers (x-forwarded-for / user-agent), never from the request body; timestamp from server clock (UTC) |
| T-56-14 | Tampering | Immutable signed archive | mitigate | New file via StorageService (no overwrite/delete path); forensic row is write-once (D-06); re-sign blocked by 409 (D-08) |
| T-56-15 | Injection | Signature PNG bytes | mitigate | base64 prefix stripped server-side + PNG magic-byte validation + 1MB size guard before embedding |
| T-56-16 | Denial of Service | Large signature payload | mitigate | 2mb body cap (Plan 02) + per-buffer 1MB PNG guard + F53 strict throttle on portal routes |
</threat_model>
