## Critical Gaps (Blockers)

### GAP-1: SECRETARIA cannot log contacts (null profesionalId FK)

**Affected:** LOG-01, ACCION-03
**File:** `backend/src/modules/pacientes/pacientes.controller.ts:109`

```typescript
const profesionalId = req.user?.profesionalId as string;
// For SECRETARIA/FACTURADOR: profesionalId is null in JWT payload
// ContactoLog.profesionalId is non-nullable → Prisma FK error at runtime
```

SECRETARIA is explicitly allowed for `/dashboard/accion` and the patient drawer, so this is a real user-facing bug.

**Fix options:**
- **(A) Resolve from patient** — look up `paciente.profesionalId` as the FK when caller has no `profesionalId`
- **(B) Require `profesionalId` body param for SECRETARIA** — front-end passes the patient's assigned professional
- **(C) Restrict endpoint to ADMIN/PROFESIONAL only** — update permissions, remove SECRETARIA from `/dashboard/accion`
