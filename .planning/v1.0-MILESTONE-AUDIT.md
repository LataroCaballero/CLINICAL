## Integration Findings

### CRITICAL — MISSING-1: SendWAMessageModal email channel always 404

**File:** `frontend/src/components/whatsapp/SendWAMessageModal.tsx:58`

Email channel calls `POST /pacientes/:pacienteId/enviar-email` — this endpoint does not exist. The correct backend email endpoint is `POST /presupuestos/:id/enviar-email` (PresupuestosController) but SendWAMessageModal has no presupuesto context.

Documented in 04-04 SUMMARY as "404 handled gracefully with toast.error" — but the email channel **always fails** from PacienteDetails, AppointmentDetailModal, and any other WA entry point.

**Fix options:**
- (A) Remove email channel from `SendWAMessageModal` — direct users to `EnviarPresupuestoModal` for email
- (B) Create `POST /pacientes/:id/enviar-email` accepting subject+body for freeform email

---

### CRITICAL — MISSING-2: WhatsappController missing SECRETARIA null-guard

**File:** `backend/src/modules/whatsapp/whatsapp.controller.ts` (all endpoints)

All WA endpoints use `req.user.profesionalId as string` without null-guard. SECRETARIA JWT has `profesionalId = null`:
- Send/thread/retry endpoints → NotFoundException from `getDecryptedConfig(null)`
- `getUnreadCounts(null)` → empty object silently → global badge shows 0 (false "all clear" for SECRETARIA)

Phase 02.1 fixed the identical bug in `pacientes.controller.ts`. Same pattern needed here.

---

### HIGH — MISSING-3: BACKEND_URL not documented; WA presupuesto fails in production

**File:** `backend/src/modules/whatsapp/whatsapp.controller.ts:131-134`

```typescript
const pdfPublicUrl = `${process.env.BACKEND_URL ?? process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:3001'}/presupuestos/public/${presupuesto.id}/pdf`;
```

`BACKEND_URL` not documented anywhere. In production defaults to `localhost:3001` — Meta cannot fetch the PDF document, all WA presupuesto sends fail at Meta delivery.

**Fix:** Document `BACKEND_URL=https://api.domain.com` in CLAUDE.md and any `.env.example`.
